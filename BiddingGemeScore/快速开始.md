# 快速开始指南

## 一分钟上手

### 1. 准备数据文件

创建CSV文件 `my_requirements.csv`：

```csv
需求名称,业务线,需求分类,季度计划,启动状态,是否故障,关联重点项目数
需求A,考研,考核落地项,本季度,开发中,否,0
需求B,考研,重点项目,本季度,开发中,否,2
需求C,四六级,其他需求,下季度,尚未启动,否,0
```

### 2. 运行脚本

```bash
python3 requirement_scorer.py -i my_requirements.csv -t 1000 -o output/my_report.md
```

### 3. 查看结果

打开 `output/my_report.md` 查看生成的报告！

## 常用命令

```bash
# 使用CSV文件
python3 requirement_scorer.py -i requirements.csv -t 1000

# 使用JSON文件
python3 requirement_scorer.py -i requirements.json -t 1000

# 指定输出路径
python3 requirement_scorer.py -i requirements.csv -t 1000 -o reports/本月排期.md

# 运行测试示例
python3 quick_test.py
```

## 数据格式要求

### CSV格式（推荐）

| 字段名 | 说明 | 可选值 |
|--------|------|--------|
| 需求名称 | 需求的名称 | 任意文本 |
| 业务线 | 所属业务线 | 考研/四六级/专升本/轻学 |
| 需求分类 | 需求类型 | 考核落地项/重点项目/其他需求 |
| 季度计划 | 计划季度 | 本季度/下季度/未进入 |
| 启动状态 | 开发状态 | 开发中/尚未启动 |
| 是否故障 | 是否X类需求 | 是/否 |
| 关联重点项目数 | B类需求关联数 | 数字（默认0） |
| 时间维度 | 时间维度（轻学必填） | 月度/季度 |

### JSON格式

```json
{
  "total_score": 1000,
  "requirements": [
    {
      "name": "需求名称",
      "business_line": "考研",
      "category": "考核落地项",
      "quarter_plan": "本季度",
      "status": "开发中",
      "is_fault": false,
      "related_projects": 0
    }
  ]
}
```

## 打分规则速查

### 优先级得分 = A × B × C

**A (需求分类)**:
- 考核落地项 = 10分
- 重点项目 = 3分 + 关联重点项目数
- 其他需求 = 1分

**B (季度计划)**:
- 本季度 = 3分
- 下季度 = 2分
- 未进入 = 1分

**C (启动状态)**:
- 开发中 = 5分
- 尚未启动 = 1分

### 资源配额

- 考研: 60%
- 四六级: 20%
- 专升本: 20%

**回流机制**: 如果某业务线无需求，其配额自动转给考研。

**轻学规则**:
- 轻学是考研的一部分，轻学需求保证入选
- 月度轻学：20分封顶
- 季度轻学：60分封顶
- 轻学配额从考研配额中预留

## 常见问题

**Q: 如何标记X类需求（故障）？**
A: 在CSV中将"是否故障"列设置为"是"，或在JSON中将`is_fault`设置为`true`。

**Q: 如何设置关联重点项目数？**
A: 仅对"重点项目"（B类）有效，在"关联重点项目数"列填写数字。

**Q: 报告保存在哪里？**
A: 默认保存在 `output/report.md`，可通过 `-o` 参数指定路径。

**Q: 支持哪些文件格式？**
A: 支持CSV和JSON格式，脚本会自动检测文件类型。
