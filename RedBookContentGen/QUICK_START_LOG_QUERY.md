# 日志查询功能快速开始

## 1. 启动 Web 应用

```bash
python3 web_app.py
```

## 2. 访问日志查询页面

在浏览器中打开以下任一地址：

- 方式 1：直接访问 http://localhost:8080/logs
- 方式 2：访问主页 http://localhost:8080，点击右上角的"日志查询"链接

## 3. 查看日志统计

页面顶部显示四个统计卡片：
- 总日志数
- 错误数（红色）
- 警告数（黄色）
- 今日日志（蓝色）

## 4. 过滤日志

### 按级别过滤
1. 在"日志级别"下拉框中选择级别（如 ERROR）
2. 点击"查询"按钮

### 按来源过滤
1. 在"日志来源"下拉框中选择模块（如 content_generator）
2. 点击"查询"按钮

### 按时间范围过滤
1. 设置"开始时间"和"结束时间"
2. 点击"查询"按钮

### 搜索关键词
1. 在"关键词搜索"框中输入关键词（如 "生成"）
2. 按回车或点击"查询"按钮

### 组合查询
可以同时使用多个过滤条件，例如：
- 级别：ERROR
- 来源：image_generator
- 关键词：API

## 5. 浏览日志

- 日志按时间倒序排列（最新的在前）
- 每页显示 50 条日志
- 使用底部的分页控件翻页

## 6. 重置过滤

点击"重置"按钮清除所有过滤条件，查看所有日志

## 常见使用场景

### 场景 1：查找最近的错误
1. 选择级别：ERROR
2. 点击"查询"
3. 查看最新的错误日志

### 场景 2：调试特定模块
1. 选择来源：content_generator
2. 点击"查询"
3. 查看该模块的所有日志

### 场景 3：分析今天的问题
1. 设置开始时间：今天 00:00
2. 设置结束时间：现在
3. 选择级别：ERROR 或 WARNING
4. 点击"查询"

### 场景 4：搜索特定操作
1. 输入关键词：图片生成
2. 点击"查询"
3. 查看所有相关日志

## 提示

- 💡 使用关键词搜索可以快速定位问题
- 💡 组合多个过滤条件可以精确查询
- 💡 定期查看错误日志，及时发现问题
- 💡 使用时间范围过滤可以提高查询速度
