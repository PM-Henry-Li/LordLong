# 日志查询界面截图说明

## 界面概览

日志查询界面采用现代化的设计风格，提供直观的用户体验。

### 1. 顶部导航栏

- 左侧：日志查询标题和图标
- 右侧：返回首页链接

### 2. 统计卡片区域

显示四个关键指标：
- **总日志数**：所有日志记录的总数
- **错误数**（红色）：ERROR 和 CRITICAL 级别的日志数量
- **警告数**（黄色）：WARNING 级别的日志数量
- **今日日志**（蓝色）：今天产生的日志数量

### 3. 过滤面板

提供多种过滤选项：
- **日志级别**：下拉选择（DEBUG、INFO、WARNING、ERROR、CRITICAL）
- **日志来源**：下拉选择（自动从日志文件中提取）
- **开始时间**：日期时间选择器
- **结束时间**：日期时间选择器
- **关键词搜索**：文本输入框，支持回车搜索

操作按钮：
- **重置**：清除所有过滤条件
- **查询**：执行搜索

### 4. 日志列表区域

每条日志记录显示：
- **日志级别标签**：带颜色标识
  - DEBUG：蓝色
  - INFO：浅蓝色
  - WARNING：黄色
  - ERROR：红色
  - CRITICAL：深红色
- **时间戳**：格式化的日期时间
- **日志来源**：模块名称标签
- **日志消息**：完整的日志内容
- **详细信息**：模块、函数、行号、线程信息
- **异常信息**（如果有）：带红色边框的异常详情

### 5. 分页控件

- 上一页/下一页按钮
- 页码按钮（当前页高亮显示）
- 省略号（当页数较多时）

## 颜色方案

- **DEBUG**：`#e8f4fd` 背景，`#0969da` 文字
- **INFO**：`#ddf4ff` 背景，`#0969da` 文字

- **ERROR**：`#ffebe9` 背景，`#cf222e` 文字

- **主色调**：`#ff2442`（小红书红）
- **背景色**：`#f5f5f7`（浅灰）
- **卡片背景**：`#ffffff`（白色）
- **文字主色**：`#1d1d1f`（深灰）
- **文字次色**：`#6e6e73`（中灰）

## 交互效果

### 悬停效果

- 日志条目：背景变为 `#fafafa`
- 按钮：轻微上移，添加阴影
- 导航链接：背景变为浅红色，文字变为主色调

### 加载状态

- 显示旋转的加载动画
- 提示文字："加载日志中..." 或 "查询中..."

### 空状态

- 显示大图标（半透明）
- 提示文字："没有找到日志" 或 "尝试调整筛选条件"

## 响应式设计

### 桌面端（> 768px）

- 统计卡片：4 列网格布局
- 过滤面板：多列网格布局
- 日志列表：完整宽度

### 移动端（≤ 768px）

- 统计卡片：单列布局
- 过滤面板：单列布局
- 日志列表：完整宽度，字体适当缩小

## 使用提示

### 快速查找错误

1. 在"日志级别"下拉框中选择"ERROR"
2. 点击"查询"按钮
3. 查看所有错误日志

### 查找特定模块的日志

1. 在"日志来源"下拉框中选择模块名称（如 "content_generator"）
2. 点击"查询"按钮
3. 查看该模块的所有日志

### 查找特定时间段的日志

1. 设置"开始时间"和"结束时间"
2. 点击"查询"按钮
3. 查看该时间段内的所有日志

### 搜索关键词

1. 在"关键词搜索"框中输入关键词（如 "生成"）
2. 按回车或点击"查询"按钮
3. 查看包含该关键词的所有日志

### 组合查询

可以同时使用多个过滤条件，例如：
- 级别：ERROR
- 来源：image_generator
- 关键词：API
- 时间范围：今天

这样可以精确定位到特定的问题日志。

## 性能优化建议

1. **使用过滤条件**：避免一次性加载所有日志
2. **限制时间范围**：查询特定时间段的日志
3. **定期清理日志**：删除或归档旧日志文件
4. **使用关键词搜索**：快速定位特定问题

## 常见问题

### Q: 为什么查询很慢？

A: 日志文件较大时，查询需要读取整个文件。建议：
- 使用过滤条件缩小查询范围
- 定期清理旧日志
- 考虑使用日志数据库（如 Elasticsearch）

### Q: 为什么有些日志显示不完整？

A: 可能是日志格式不正确。确保日志文件使用 JSON 格式，每行一个 JSON 对象。

### Q: 如何导出日志？

A: 目前不支持导出功能。可以：
- 直接复制日志内容
- 使用浏览器的打印功能保存为 PDF
- 未来版本将添加导出功能

### Q: 如何查看实时日志？

A: 目前不支持实时日志流。需要刷新页面查看最新日志。未来版本将添加 WebSocket 实时推送功能。
